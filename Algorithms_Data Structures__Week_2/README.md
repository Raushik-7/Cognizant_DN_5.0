
# Algorithms_Data_Structures
This repository contains solutions for Week-2 exercises as part of the Cognizant learning program.

## Exercise 1 : E-commerce Platform Search Function

### Objective
Implement Linear Search and Binary Search to find products in an e-commerce platform and compare their efficiency.

### Concepts Used
- Arrays
- Linear Search
- Binary Search
- Sorting
- Time Complexity Analysis

### Features
- Store product details using a Product class.
- Search products using Linear Search.
- Search products using Binary Search.
- Compare search performance.

### Time Complexity

| Algorithm | Best Case | Average Case | Worst Case |
|------------|-----------|-------------|------------|
| Linear Search | O(1) | O(n) | O(n) |
| Binary Search | O(1) | O(log n) | O(log n) |

### Conclusion
Binary Search is more efficient for large datasets because it reduces the search space by half in every iteration. However, it requires the data to be sorted before searching.

## Exercise 2 : Financial Forecasting

### Objective
Implement a recursive algorithm to forecast the future value of an investment based on a fixed annual growth rate.

### Concepts Used
- Recursion
- Mathematical Computation
- Time Complexity Analysis

### Features
- Calculates future investment value recursively.
- Demonstrates the use of a base case and recursive calls.
- Displays the forecasted value after a specified number of years.

### Time Complexity

- **Time:** O(n)
- **Space:** O(n)

### Optimization
The recursive approach is simple and easy to understand. For larger inputs, an iterative approach can reduce the space complexity to **O(1)**.
